## Abstract
This HIP establishes the registry entry for Hiero Consensus Standards (HCS). It links to the canonical specifications in the `hiero-consensus-specifications` repository and pins the current published versions to a specific commit for reproducibility. No protocol behavior changes are introduced; this is a documentation and traceability artifact.

## Motivation
The Hiero Consensus Standards have been migrated to `hiero-consensus-specifications`. The TSC requested a stable, reviewable pointer in the HIPs repository so downstream consumers, auditors, and tooling can track which specification versions are current. Recording the canonical source and commit hash here reduces ambiguity and aligns lifecycle management with the process defined in HCS-4.
